## üìù Writing Conventions

### Code Examples
All documentation uses proper code formatting with language identifiers:
```php
// PHP code
```

```bash
# Bash commands
```

```javascript
// JavaScript
```

### File References
Files are referenced with proper paths:
- `app/Models/User.php`
- `resources/views/home/index.blade.php`
- `.env` for environment files

### Commands
Terminal commands are prefixed with appropriate shell:
- `php artisan ...` for Laravel commands
- `npm run ...` for Node commands
- `composer ...` for Composer commands

### Links
Documentation links use relative paths:
- `[Documentation](DOCUMENTATION.md)`
- `[Setup Guide](SETUP_GUIDE.md)`
